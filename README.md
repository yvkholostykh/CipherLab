🟢 6. Описание алгоритмов
🔵 6.1 Простая замена

📌 Используется фиксированный (постоянный) алфавит подстановки.
📌 Ввод ключа вручную не требуется.
📌 Каждая буква заменяется соответствующей буквой шифралфавита.

Преимущества:
✔ Простота реализации
✔ Наглядность

Недостатки:
❌ Легко взламывается частотным анализом

🔵 6.2 Аффинный шифр

Используется формула:

Y = (aX + b) mod m

где:

X — номер буквы

a, b — ключевые параметры

m = 33 (размер русского алфавита)

Условие корректности:

gcd(a, m) = 1

Преимущества:
✔ Более сложный, чем простая замена

Недостатки:
❌ Всё ещё уязвим к криптоанализу

🔵 6.3 Рекуррентный аффинный шифр

Коэффициенты изменяются для каждой буквы:

a_i = (a_{i-1} * a_{i-2}) mod m
b_i = (b_{i-1} + b_{i-2}) mod m

Преимущества:
✔ Более высокая криптостойкость
✔ Сложнее взламывается

🟢 7. Режим работы

После выбора алгоритма программа спросит:

D - дешифрование

Введите:

E — чтобы зашифровать текст

D — чтобы расшифровать

🟢 10. Возможные ошибки
❌ "Ключ должен содержать 33 буквы"

Возникает при неправильном вводе ключа (в старой версии).

❌ "a не взаимно просто с 33"

Нужно выбрать другое значение a.

❌ SyntaxError

Проверьте структуру программы и отступы.
